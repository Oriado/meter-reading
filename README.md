# Розпізнавання показників газових лічильників

Веб-додаток для автоматичного зчитування цифр з фото газового лічильника. Завантажуєш фото — програма знаходить панель з цифрами, вирівнює її і зчитує показник.

## Як це працює

```
Фото лічильника
      ↓
YOLO (знаходить 4 кути панелі з цифрами)
      ↓
Perspective Transform (вирівнює перекошене фото)
      ↓
CRNN (зчитує всі цифри одразу)
      ↓
Числовий показник
```

**YOLO** — знаходить не просто прямокутник, а саме 4 кути (keypoints): лівий верхній, правий верхній, правий нижній, лівий нижній. Це дозволяє вирівняти фото зроблене під кутом.

**CRNN** — це CNN + двонаправлений GRU + CTC Loss. CNN витягує ознаки із зображення, GRU читає послідовність зліва направо і справа наліво одночасно, CTC дозволяє розпізнавати рядок без сегментації на окремі цифри.

## Файли

- `app.py` — Streamlit додаток
- `yolo_pose_best.pt` — навчена YOLO модель (keypoint detection)
- `ocr_model.pt` — навчена CRNN модель
- `yolo.ipynb` — навчання YOLO
- `ocr_model.ipynb` — навчання CRNN
- `transform.ipynb` — perspective transform
- `pose_data.yaml` — конфіг датасету
- `metrics/` — метрики моделей у JSON

## Запуск

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Дані

Датасет: 2807 фото лічильників з Roboflow (2188 train / 412 val / 207 test).

Для навчання YOLO keypoints — кути панелі з цифрами розмічені вручну як 4 точки.

CRNN навчалась на синтетичних зображеннях цифрових рядків, після чого донавчалась на реальних кропах газових лічильників.

## Метрики

**YOLO:** mAP50 = 0.874, Precision = 0.777, Recall = 0.766, навчалась 50 епох.

**CRNN:** val accuracy = 100% на синтетичних даних, навчалась 30 епох на 5000 зображеннях.

## Обмеження

Модель навчена на одному типі газового лічильника (Itron GALLUS 2000). На інших марках точність нижча — різний шрифт цифр і розташування панелі. Для покращення потрібно більше різноманітних фото з розміткою.
